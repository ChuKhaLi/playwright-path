Performance Considerations

Selector Efficiency Guidelines

Browsers parse CSS selectors from right to left. Here are examples of efficient vs. inefficient patterns:

✅ Efficient Selectors:
.button { }
.nav-link { }
[data-component="price"] { }
.sidebar .widget-title { }
❌ Avoid These Patterns:
* * * .button { /* Too many universal selectors */ }
.container .main .sidebar .widget .title { /* Too deeply nested */ }
div:has(div:has(div:has(.target))) { /* Nested :has() chains */ }

Browser Support Notes

Modern Features Compatibility

  • :has() selector - Supported in Chrome 105+, Firefox 103+, Safari 15.4+
  • :is() and :where() - Good support across modern browsers
  • :nth-child() with complex formulas - Excellent support
  • Advanced attribute selectors - Excellent support

Always test advanced selectors in your target browsers, especially :has() for production use.
